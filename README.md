## About This Portfolio

이 포트폴리오는 제가 참여한 프로젝트와  
실무 경험을 정리한 사이트입니다.

단순 결과물이 아닌,  
**어떤 구조로 설계했고, 어떤 문제를 해결했는지**에 집중했습니다.

프레임워크 기반 화면 개발 경험을 바탕으로  
퍼블리싱을 넘어 **구조적인 UI 설계와 구현까지 확장**해온 과정을 담았습니다.

---

## What I Focus On

- UI 구현 이전에 **구조와 흐름을 먼저 설계**
- 반복 UI를 줄이기 위한 **공통 컴포넌트 구조화**
- 상태 흐름과 데이터 구조를 고려한 **화면 설계**
- 유지보수를 고려한 **SCSS 구조 및 스타일 관리**
- 인터랙션은 단순 효과가 아닌 **사용자 경험 개선 목적**

## Key Experience

- 그룹웨어, 내부 시스템, 공공 서비스, 이커머스 등 다양한 도메인의 UI 구축 경험
- 프레임워크 기반 환경에서 컴포넌트 단위 화면 구성 및 상태 관리 구조 이해
- REST API 기반 데이터 연동 UI 구현 경험
- 권한별 화면 분기, 데이터 중심 UI, 대시보드형 화면 구현 경험

## 📂 Project Structure

```
src/
 ├── components/     // 재사용 UI 컴포넌트
 ├── pages/          // 라우팅 단위 페이지
 ├── data/           // 프로젝트 데이터 관리
 ├── assets/         // 이미지 및 정적 리소스
 └── styles/         // SCSS 구조 관리
```
